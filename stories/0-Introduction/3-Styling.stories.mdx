# Styling

There are 4 main aspects to consider when we talk about styling with respect to the Quark Web Components, all of which are based on Material Design.

* Themes
* Setting Colors
* Typography
* Elevation

Themes are used to specify a predefined set of colors (exposed as CSS custom variables) that are used to set various colors in UI elements. Since they use CSS custom variables, switching themes will result in all elements that use the CSS custom variables to be automatically updated with the new color value. Typography is controlled *via* a predefined set of font styles for most of the font styles defined by Material Design (a few are omitted because they aren't necessary for line of business applications). Finally, Material Design makes extensive use of 'elevation' to give the impression of raised surfaces, which is exposed as a set of CSS custom variables to set different drop-shadow heights. The sections that follow cover these topics in more detail.

## Themes

Theming only involves colors and follows a modified version of the [Material Design standards](https://material.io/design/color/the-color-system.html#color-theme-creation) (a mix of V1 and V2) and is implemented as a fairly large set of CSS custom variables, of which only a few are configurable.

### Creating Themes

The theming system only has a single default theme (blue and pink), so you'll probably want to create at least one additional theme for your application. Theme management is all done using `windows.ThemeManager`, which is initialized by simply importing the theme manager module.

```javascript
import 'quark-web-components/theme/theme-manager.js';
```

You can then create a theme like this:

```javascript
import { Theme } from 'quark-web-components/theme/theme.js';

window.ThemeManager.addTheme(new Theme('green', '#80e27e', '#4caf50', '#087f23', '#ffc107'));
```

You can also add multiple themes using the `addThemes` method, which takes an array of `Theme` instances.

The `Theme` constructor has the following signature (the parameters are self-explanatory):

```javascript
// Color values must be provided as CSS color value strings.
constructor(name, primaryLight, primary, primaryDark, accent);
```
The default theme has the name `default` and is always available, but you can remove it or overwrite it by adding a new theme with the same name.

### Setting Themes
Setting themes should be restricted to specific elements (usually the top level application element) and is managed *via* `window.ThemeManager`. After adding themes, you can set the active theme at the document level like this:

```javascript
window.ThemeManager.theme = 'green';

// or

window.ThemeManager.applyTheme('green');
```

Themes can also be set at an HTML element level, which allows different themes to be applied to different elements (and their children) and is done by using the `applyTheme` method. The `applyTheme` method takes a theme name and the element to apply the theme to. If no element is provided then the theme is applied to the document (as shown above).

```javascript
window.ThemeManager.applyTheme('green', el);
```

You can clear a theme from an element using the `clearTheme` method, which takes the element to clear as the only parameter.

```javascript
window.ThemeManager.clearTheme(el);
```

You can delete a theme using the `deleteTheme` method, which takes the theme name as the only parameter. Deleting a theme will also clear it from the document element, if it's the active theme.

```javascript
window.ThemeManager.deleteTheme('green');
```

## Setting Colors

Using the theme colors is trivial, you just use the appropriate CSS custom variables wherever necessary.

```css
.some-class {
    color: var(--theme-on-primary-color);
    background: var(--theme-primary-color);
    border: 1px solid var(--theme-divider-on-primary-color);
}
```

This will work with any JavaScript application framework or library.

## Typography

A simplified version of the Material Design guidelines for typography is used, which is mostly based on V1, with a bit from V2 (overline) and the addition of a 'code' (monospace) style. To use the styles you should use the following [Google Web Fonts](https://fonts.google.com/) in your main document:

### Font Importing

The predefined font styles depend on the use of the Roboto (normal and monospace) and Noto fonts. You should always use these fonts as other fonts will probably give inconsistent styling.

#### Roboto

Use just the Roboto font if you don't need monospace or simplified Chinese support.

```html
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;500&display=swap" rel="stylesheet">

<!-- OR -->

<style>
    @import url('https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;500&display=swap');
</style>
```

Use the following CSS rule:

```css
font-family: 'Roboto', sans-serif;
```

#### Roboto + Noto

If you need simplified Chinese support, use the following:

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500&family=Roboto:wght@300;400;500&display=swap" rel="stylesheet">

<!-- OR -->

<style>
    @import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500&family=Roboto:wght@300;400;500&display=swap');
</style>
```

Use the following CSS rule:

```css
font-family: 'Roboto', sans-serif;
font-family: 'Noto Sans SC', sans-serif;
```

#### Roboto + Roboto Mono + Noto

If you need simplified Chinese support and Roboto monospace (for code samples), use the following:

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500&family=Roboto+Mono&family=Roboto:wght@300;400;500&display=swap" rel="stylesheet">

<!-- OR -->

<style>
    @import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500&family=Roboto+Mono&family=Roboto:wght@300;400;500&display=swap');
</style>
```

Use the following CSS rule:

```css
font-family: 'Roboto', sans-serif;
font-family: 'Noto Sans SC', sans-serif;
font-family: 'Roboto Mono', monospace;
```
*You can obviously mix and match the Roboto and Mono fonts however you need.*

### Usage

How you use the font styles depends on which JavaScript framework or library you're using. Currently, the following are supported:

* LitElement

#### LitElement

To use the typography styles, you first need to import the style classes you want to use.

```javascript
// Import individual styles.
import { body, caption } from 'quark-web-components/styles/typography.js';

// Import all the styles.
import { typography } from 'quark-web-components/styles/typography.js';
```

Then add the typography style imports to your element static style getter (don't forget to do this):

```javascript
static get styles() {
    return [body, caption, css`
        :host {
            display: block;
        }
    `];
}

// OR

static get styles() {
    return [typography, css`
        :host {
            display: block;
        }
    `];
}
```

Finally, use the style classes to your markup (the class name is the module import name with a `typography-` prefix):

```javascript
render() {
    return html`
        <div class="typography-body">Body</div>
        <div class="typography-caption">Caption</div>
    `;
}
```

## Elevation

Material Design uses drop shadows to simulate elevation (raised elements). This is supported *via* LitElement CSS module exports that exports CSS classes for each elevation height that can either be applied directly the a custom element (styles :host) or elements within an element. The 0, 1, 2, 3, 4, 6, 8, 12, 16 and 24 pixel height elevations are supported.

<div class="elevation-container">
    <div class="elevation-0">0</div>
    <div class="elevation-1">1</div>
    <div class="elevation-2">2</div>
    <div class="elevation-3">3</div>
    <div class="elevation-4">4</div>
    <div class="elevation-6">6</div>
    <div class="elevation-8">8</div>
    <div class="elevation-12">12</div>
    <div class="elevation-16">16</div>
    <div class="elevation-24">24</div>
</div>

### Usage

How you use the elevation styles depends on which JavaScript framework or library you're using. Currently, the following are supported:

* LitElement

#### LitElement

The method for styling the host or child elements is slightly different.

##### Host Elevation

The host elevation imports are named `hostElevation{x}`, where `x` is the elevation height and adds a class with a `:host([elevation="x"])` selector to the styles. You set the elevation by simply adding an `elevation` attribute with the appropriate value to the element when you use it.

Firstly, import the host elevations you want to use, either individually or the whole lot.

```javascript
// Import individual host elevations.
import { hostElevation1, hostElevation2 } from 'quark-web-components/styles/elevation.js';

// Import all the host elevations.
import { hostElevation } from 'quark-web-components/styles/elevation.js';
```

Then add the host elevation style imports to your element static style getter (don't forget to do this):

```javascript
static get styles() {
    return [hostElevation1, hostElevation2, css`
        :host {
            display: block;
        }
    `];
}

// OR

static get styles() {
    return [hostElevation, css`
        :host {
            display: block;
        }
    `];
}
```

Now, consumers of your element can apply an elevation by using the `elevation` attribute.

```html
<my-element elevation="1"></my-element>
<my-element elevation="2"></my-element>
```

##### Child Elevation

The child elevation imports are named `elevation{x}`, where `x` is the elevation height and adds a class called `elevation-{x}` to the element styles. You set the elevation by simply adding the `elevation-{x}` class to the child element that needs an elevation.

Firstly, import the child elevations you want to use, either individually or the whole lot.

```javascript
// Import individual child elevations.
import { elevation1, elevation2 } from 'quark-web-components/styles/elevation.js';

// Import all the child elevations.
import { elevation } from 'quark-web-components/styles/elevation.js';
```

Then add the child elevation style imports to your element static style getter (don't forget to do this):

```javascript
static get styles() {
    return [elevation1, elevation2, css`
        :host {
            display: block;
        }
    `];
}

// OR

static get styles() {
    return [elevation, css`
        :host {
            display: block;
        }
    `];
}
```

Finally, use the `elevation-{x}` classes.

```javascript
render() {
    return html`
        <div class="elevation-1">Elevation 1</div>
        <div class="elevation-2">Elevation 2</div>
    `;
}
```
